> Dynamic Real-time Interactive Vehicle Environment - Honda (USA)


Graduate Research Assistantship at Georgia Tech with the Aerospace Systems Design Laboratory – Sponsored project for Honda (USA)

Mechanical Simulation Engineer


Project Summary

Developed real-time driver-in-the-loop vehicle simulation models using SIMPACK by Dassault Systèmes. Modulated system model fidelity of multi-body dynamics components and proposed alternatives to balance performance and cost.


Why the Automotive Industry Needs Real-Time (RT) Simulation

The automotive industry depends on a diverse range of tests to develop production-ready vehicles that meet high standards of safety, performance, comfort, durability, and regulatory compliance. Both objective and subjective metrics are essential when evaluating a vehicle’s performance, comfort, and usability. Simulations, particularly those incorporating driver-in-the-loop technology, enable the collection of subjective metrics through real-time (RT) simulation. This becomes possible only if the simulation is sufficiently realistic.


What is Real-Time Simulation?

Real-time simulation refers to computer models that operate at the same rate as the physical system they are representing. This allows for dynamic, real-time interactions with the model, making it possible to simulate complex systems in realistic environments.

Real-time simulation can demand significant computational resources.

Image Source: FAAC


Objective:

Honda provided us with a high-fidelity car model developed in Simpack.

The objective is to streamline the process of creating and refining the fidelity of component models for RT simulation. Specifically, we aim to propose a method for evaluating the trade-off between real-time computational performance and model error in multi-body dynamics (MBD) for RT driver-in-the-loop simulations.


Technical Tasks:

    • Configure the baseline model on Simpack based on Honda's car model provided.
    • Configure the solver settings.
    • Analyze simulation bottlenecks to identify opportunities for simplifying the baseline model's components.
    • Generation of all alternative simulations with varying levels of fidelity using statistical techniques such as Design of Experiments (DOE).
    • Run simulations using Simpack.
    • Determine the real-time factor (computational cost) and quantify the error for each simulation.
    • Rank the alternatives by evaluating the trade-off between computational cost and performance.

Model Simplification

The key components of focus include suspensions, subframe, tires, and steering.

Each alternative represents a variation of our car model, where different components have specific fidelity levels that vary across simulations.

Example of component simplification:

Additionally, we will employ Design of Experiments (DoE) to systematically analyze how different factors affect the outcome. DoE will help us plan, conduct, and interpret controlled tests to evaluate the relationships between process factors and model outputs.


Main Tools and Enablers:

  • JMP: Used for generating DoE and creating surrogate models to predict performance across different alternatives.
  • Simpack: Used to model mechanical components and simulate their behavior in various conditions.


Error Quantification of the simulations and Challenges:

Error quantification involves measuring the difference between a simplified model and the high-fidelity car model, focusing particularly on chassis speeds and accelerations. Even small discrepancies can lead to significant objective and subjective differences perceptible to the driver.

Key challenges include:

  • Point-to-point alignment of dynamic time serie between the baseline's outputs and the simulations' outputs.
  • Complex interactions between components in the multi-body system.
  • Handling multivariate problems.

Dynamic Time Warping (DTW) will be employed to address these challenges:

  • It can handle temporal misalignments effectively.
  • It is robust to speed variations and captures similarities between different nonlinearities.
  • It doesn’t require uniform sampling, making it adaptable to real-world data.
  • DTW allows for intuitive visualizations that can aid in understanding the discrepancies between the models and real-world measurements.

> Analysis Program for Spine Kinematics - Georges Charpak Biomechanics Laboratory (Paris)


Collaborative project aimed at developing a program to analyze the kinematics of a human spine by tracking vertebrae in 3D from X-ray images. The project involved a team of four researchers and two academic advisors.

Role: Lead for Point Detection and Temporal Tracking


Project Summary

Developed an analysis tool to track and render the movement of human spine over time using X-ray images taken at various angles. The workflow included 2D point detection, point matching between views, 3D point reconstruction, clustering for vertebrae identification, and user interface development for manual intervention and result refinement, using python.


Key Methodologies

1. Point Detection

Identified the 2D positions of metallic markers in X-ray images using two main approaches:

  • OpenCV-Based Detection: Utilized SimpleBlobDetector() after pre-processing the images with adaptive thresholding to enhance the visibility of the markers.
  • Image Correlation: Compared regions of the image with a sample marker template to find matches without requiring binary image conversion. Achieved 95% accuracy in detecting markers.
2. Point Matching

Matched detected markers between two views taken at the same time step using clustering and DBSCAN methods, achieving near-perfect accuracy. Challenges included occlusions, which were addressed through clustering and point validation techniques.

3. Temporal Tracking

Tracked marker movement between consecutive time steps with two approaches:

  • OpenCV Object Tracking: Initial attempts using OpenCV’s built-in methods (e.g., brute force) were unreliable due to shape deformation in binary images.
  • Custom Algorithm: Developed an algorithm assuming planar motion (e.g., in X-Z plane) to match points based on proximity in consecutive time frames. Achieved nearly 100% accuracy.
4. Data Clustering and Visualization

Clustered 3D points to group markers by vertebrae. Challenges such as overlapping markers were addressed through a combination of DBSCAN and manual correction options in the interface.

5. User Interface

Designed a Tkinter-based interface allowing users to visualize results, adjust detection thresholds, and manually correct errors. This ensured the tool was user-friendly and adaptable for various image sets.


Error Rates and Observations

  • Image Correlation: 21% error
  • Improved Correlation: 17% error
  • Thresholding: 20% error
  • Point Matching: 2% error
  • Clustering: 5% error

The tool significantly reduced manual processing time while requiring minimal human intervention for final adjustments.

> Large-Scale Watch Mechanism - Mechanical Design and Fabrication


Collaborative project focused on designing and fabricating a functional large-scale watch mechanism using additive manufacturing and numerical simulations. The project involved a team of four engineers and two academic advisors.

Role: Lead for Resonator, Escapement and Anchor System


Project Summary

Developed a fully functional large-scale watch mechanism with a focus on precision, compactness, and manufacturability. The workflow included material characterization, CAD modeling, additive manufacturing, and iterative testing.

Image Source: FAAC


How a Mechanical Watch Works: The Role of Key Components

A mechanical watch works through a complex series of interactions between its key components. The main power source is the mainspring, which stores energy when wound. This energy is transferred through the gear train, which regulates the movement of the watch hands. The escapement controls the release of energy, ensuring a consistent flow to the balance wheel, which oscillates back and forth, regulating the watch's timekeeping. The crown is used to wind the mainspring and set the time. Together, these components work in harmony, allowing the watch to keep accurate time without the need for batteries.


Key Methodologies

1. Mainspring Barrel Design

Designed a mainspring capable of storing and delivering consistent energy, optimizing autonomy, and the associated cliquet.

  • Material Selection: PLA with a yield strength of 49.5 MPa.
  • Geometric Optimization: Achieved an autonomy of 14.5 turns.
2. Gear Train Optimization

Designed an efficient gear train to achieve accurate timekeeping, comparing planetary and parallel-axis configurations.

  • Reduction Ratios: Optimized using analytical and simulation techniques.
  • Final Decision: Chose parallel-axis gears due to manufacturability and reduced friction.
3. Resonator and Escapement System

Designed a balance wheel and escapement system for accurate frequency control.

  • Oscillation Frequency: Targeted 2 Hz for optimal performance.
  • Fine-Tuning: Adjustable regulation using a raquette pin.
4. Additive Manufacturing and Testing

Used FDM printing for rapid prototyping, iterating on multiple designs to ensure proper fit and function.

  • Optimized gear tolerances to minimize backlash.
  • Designed adjustable mounting structures for modular assembly.


Error Rates and Observations

  • Gear Train: 8% error in initial trials, reduced to 2% after adjustments.
  • Resonator: Initial 3.5 Hz frequency, adjusted to 2 Hz through fine-tuning.
  • Escapement: Required multiple iterations due to high friction losses.

The final prototype provided valuable insights into watch design and manufacturing constraints.


Challenges

  • Minimizing friction between components to enhance efficiency.
  • Optimizing space management within the compact watch mechanism.
  • Ensuring the structural integrity of components (e.g., preventing plastic deformations).
  • Maintaining tight tolerances for precise assembly and functionality.
  • Coordinating teamwork on separate components that must integrate seamlessly.
  • Managing energy transfer efficiently between different parts of the watch.
  • Finding an optimal oscillation frequency while ensuring compatibility with component dimensions, particularly gear sizing and integration.

> Aircraft Brake System - Design and Analysis


Individual project focused on designing and analyzing a braking system for the rear wheels of a given light aircraft. The project involved mechanical design, structural analysis, and performance validation.


Project Summary

Developed a braking system with emphasis on efficiency, structural integrity, and manufacturability. The workflow included system selection, force calculations, CAD modeling, and verification through analytical methods. We have chosen to work with a floating caliper braking system, featuring one disc per rear wheel, with feasibility yet to be demonstrated.


Brake System Components and Functionality

The brake system operates through a hydraulic disc mechanism, applying friction to slow down the aircraft. Key components include the rotor disc, caliper, pistons, and brake pads. The hydraulic system transmits force efficiently to engage the brake mechanism.


Key Methodologies

1. Brake Disc Dimensioning

Designed the brake discs to withstand calculated thermal and mechanical loads.

  • Material Selection: Steel (7800 kg/m³).
  • Geometric Optimization: 32mm inner diameter, 150mm outer diameter.
2. Brake Force and Torque Calculation

Derived required braking torque using aircraft deceleration parameters.

  • Braking Torque: 204 N.m per wheel.
  • Surface Contact Analysis: Optimized pad-to-disc friction interface.
3. Structural Design of Caliper and Pistons

Designed the brake caliper for efficient force distribution.

  • Piston Diameter: Optimized for fluid pressure distribution.
  • Material Considerations: Lightweight yet strong alloy used for caliper.
4. Transmission and Power Assembly

Developed the shaft and fastening mechanisms ensuring durability.

  • Clavette and Cannelure: Used for rotational force transmission.
  • Fastening: M12 bolts ensuring secure assembly.


Final Result


Drafting

The complete drafting and assembly process plan for the system were carried out.

Challenges

  • Integration into a real system (considering constraints).
  • Estimation of the energies involved.
  • Minimizing heat buildup to prevent brake fade.
  • Ensuring coherence of chosen dimensions (e.g., preventing the aircraft from rolling on the brake disc in case of a tire puncture).
  • System assembly feasibility.
  • Preventing component wear (e.g., pad indentation).
  • Reducing weight while maintaining strength.
  • Ensuring appropriate functional clearances for optimal system performance.
